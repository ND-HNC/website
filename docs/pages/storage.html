Storage

This page is under construction.

Overview

To support diverse data management and computational needs, the ND-HNC – in partnership with the Center for Research Computing (CRC) – provides researchers with a shared, multi-tiered data storage infrastructure.

These resources are categorized into three distinct tiers based on the data’s activity level and backup requirements:

Storage Tier Total Capacity Mount Location Primary Purpose Backup Policy
1: Active Storage 50 TB /groups/hnc Intermediate storage of publication-critical files. Nightly
2: Cold Storage 250 TB /goldfs/hnc Long-term storage srcdata, rawdata, publication files. Nightly
3: Scratch 500 TB TBD Temporary workspace for intermediate generation. None

Key Guidelines for Users:

  • Shared Resources: Because these capacities are shared across the ND-HNC community, please manage your allocations responsibly and migrate data to the appropriate tier as your project phases change.
  • Data Safety Warning: Tier 3 (Scratch) is intended only for transient data. Do not store critical or irreplaceable files here, as they cannot be recovered in the event of a system failure.

Access

Any CRC user account that is part of the HNC group has full access to these resources. Contact Nathan Muncy to be added to the HNC group.

Dataflow

Using storage tiers for their intended use is critical when multiple users are sharing common resources. Instead of dividing available resources across all ND-HNC affiliated labs, all users currently have full access to all resources. This reflects the fact that projects across labs will be in various stages at any one time.
